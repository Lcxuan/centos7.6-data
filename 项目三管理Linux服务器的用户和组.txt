Linux操作系统是多用户多任务的操作系统

用户和族群的基本概念
概念					描述
用户名				       用来标识用户的名称，可以是字母、数字组成的字符串，区分大小写
密码				        用于验证用户身份的特殊验证阿妈
用户标识（UID）		       	    用来表示用户的数字标识符
用户主目录				     用户的私人目录
登录shell					用户登录后默认使用的shell程序，默认为/bin/bash
组群					具有相同的属性的用户属于同一组群
组群标识（GID）			    用来表示组群的数字标识符

root用户的UID：系统用户的UID从1到999
普通用户的UID：如果不指定默认从1000开始
创建用户同时也会创建一个和用户同名的组群，该组群就是用户的主组群
普通组群的GID默认也是从1000开始

useradd		添加新的用户账号
		useradd [选项] username
		选项：
			-c comment						用户的注释性信息
			-d home_dir（家目录）				   指定用户的主目录
			-e expire_date（过期时间）				  禁用账号的日期，格式为YYYY-MM-DD
			-f inactive_days（过期天数）				  设置用户过期多少天后被禁用，0为马上禁用，-1为不禁用
			-g initial_group（所属组或GID）			  用户所属主组群的组群名称或者GID
			-G group-list					        用户所属的附属组群列表，多个组群之间用逗号分隔
			-m							若用户主目录不存在则创建
			-M							不创建用户主目录
			-n							不为用户创建用户私人组群
			-p passwd						加密的口令
			-r							创建UID小于500，并且不带主目录的系统账号
			-s shell						指定用户的登录shell，默认为/bin/bash
			-u UID							指定用户的UID，必须是唯一的，且大于999
			-N							不创建私人组群，会自动创建一个GID等于100
		useradd	test


userdel		删除用户
		userdel [选项] 用户名
		选项：
			-r	将用户的主目录一起删除
		userdel test

usermod		用于修改用户的属性
		usermod [选项] 用户名
		选项：
			-c		填写用户账户的备注信息
			-d -m 		参数-m与参数-d连用，可重新指定用户的家目录并自动把旧的数组转移过去
			-e		账户的到期时间，格式为YYYY-MM-DD
			-g		变更所属用户组
			-G		变更扩展用户组
			-L		锁定用户禁止等登录系统
			-U		解锁用户，允许其登录系统
			-s		变更默认终端
			-u		修改用户的UID
		usermod -g test test1	将test1加入到test组中
			
passwd		指定和修改用户账户口令
		passwd [选项] [username]
		选项：
			-l	锁定（停用）用户账户
			-u	口令解锁
			-d	将用户口令设置为空
			-f	强迫用户下次登录时必须修改口令
			-n	指定口令的最短存活期
			-x	指定口令的最长存活期
			-w	口令到期前提前警告的天数
			-i	口令过期后多少天停用账户
			-S	显示账户口令的剪短状态信息

		passwd -l user1 锁定用户
		passwd -u user1	解锁用户
		passwd -d user1 将用户的口令设置为空
			
chage		修改用户账户口令
		chage [选项] username
		选项：
			-l	列出账户口令属性的各个数值
			-m	指定口令最短存活期
			-M	指定口令最长存活期
			-W	口令要到期前提前警告的天数
			-I	口令过期后多少天停用账户
			-E	用户账户到期作废的日期
			-d	设置口令上一次修改的日期

groupadd	创建组群
			
			
groupdel	删除组群
		groupdel [选项] 组名
		groupdel testGroup

groupmod	修改组群
		groupmod [选项] 组名
		选项：
			-g gid			把组群的GID改成gid
			-n group-name		把组群的名称改为group-name
			-o			强制接受更改的组的GID为重复的号码
			
gpasswd		附属组群添加用户、删除用户
		gpasswd [选项] [用户] [组]
		选项：
		-a		把用户加入组
		-A		给组指派管理员
		-d		把用户从组中删除
		-r		取消组的密码

su			用户切换
			-完全切换到新用户，即把环境变量信息也变更为新用户的相应信息
			
sudo		更安全的切换用户
			
用户账户文件
/etc/passwd文件	：
	创建的用户账户及其相关信息都放在这个文件
	每行用":"分隔为7个域，各域的内容：
		用户名:加密口令:UID:GID:用户的描述信息:主目录:命令解释器（登录shell）
	
	用户名：用户账号名称，用户登录时所使用的用户名
	加密口令：用户口令，考虑系统的安全性，现在已经不使用该字段保存口令，使用字母来填充，真正的密码在shadow文件中
	UID：用户号，唯一表示某用户的数字标识
	GID：用户所属的私有组号，改数字对应group文件中的GID
	用户描述信息：可选的关于用户全名、用户电话等描述信息
	主目录：用户的宿主目录，用户成功登录后的默认目录
	命令解释器：用户所使用的shell，默认"/bin/bash"
	
/etc/shadow文件：
	经过加密的口令都存放在这个文件
	所有用户都有读取权限，只对root用户可读权限
	每行用":"分隔为9个域
	字段	说明
	1		用户登录名
	2		加密后的用户口令，*表示非登录用户 !!表示没设置密码
	3		从1970年1月1日起，到用户最近一次口令被修改的天数
	4		从1970年1月1日起，到用户可以更改密码天数，即最短口令存活期
	5		从1970年1月1日起，到用户必须更改密码天数，即最长口令存活期
	6		口令过期前几天提醒用户更改口令
	7		口令过期后几天账户被禁用
	8		口令被禁用的具体日期
	9		保留域，用于功能扩展
	
/etc/login.defs文件
	建立用户账户时会根据这个文件配置设置用户账户的某些选项
	MAIL_DIR	/var/spool/mail		//用户邮箱目录
	MAIL_FILE	.mail
	PASS_MAX_DAYS	99999			//账户密码最长有效天数
	PASS_MIN_DAYS	0			//账户密码最短有效天数
	PASS_MIN_LEN	5			//账户密码的最小长度
	PASS_WARN_AGE	7			//账户密码过期前提前警告的天数
	UID_MIN		1000			//用useradd命令创建账户时自动产生的最小UID值
	UID_MAX		60000			//用useradd命令创建账户时自动产生的最大UID值	
	GID_MIN		1000			//用groupadd命令创建组群时自动产生的最小GID值
	GID_MAX		60000			//用groupadd命令创建组群时自动产生的最大GID值
	USERDEL_CMD	/usr/sbin/userdel_local	//如果定义的话，将在删除用户时执行，以删除响应用户的计划作业和打印作业等
	CREATE_HOME	yes			//创建用户账户时是否为用户常见主目录
	
组群文件：
/etc/group文件：
	用于存放用户的组账户信息
	任何用户都可以读取到
	每行用":"分隔为4个域
		组群名称：组群口令（一般为空，用x占位）:GID:组群成员列表
		
/etc/gshadow文件：
	用于存放组群的加密口令、组管理员等信息
	只用root可以读取
	每行用":"分隔4个域
	组群名称:加密后的组群口令(没有就用!):组群的管理员:组群成员列表
	
常用的账户管理命令：
	vipw	用于直接对用户账户文件/etc/passwd进行编辑，默认编辑器为vi
	vigr	用于直接对组群文件/etc/proup进行编辑，默认编辑器为vi
	pwck	用于验证用户账户文件认证信息的完整性
	grpck	用于验证组群文件认证信息的完整性
	id		用于显示一个用户的UID和GID以及用户所属的组列表
			id [选项] 用户名
			id test1
	finger	可以查看用户的相关信息
			-l	以长格形式显示用户信息，默认选项
			-m	关闭以用户姓名查询账户的功能
			-s	以短格形式查看用户的信息
			-p	不显示plan（plan信息时用户主目录下的.plan等文件）
	chfn	用来修改finger显示的内容，可以修改用户的办公地址、办公电话和住宅地址等
	chsh	用来修改finger显示的内容，可以修改用户的启动shell
	whoami	用于显示当前用户的名称
	newgrp	用于转换用户的当前组到指定的主组群
